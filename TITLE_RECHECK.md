## RECHECK_TITLES: контроль актуальности объявлений

- **Что сделали:** добавили опциональную проверку совпадения `title` между каталогом и БД. При включении режима объявления с изменившимся заголовком удаляются из таблиц `ebay`/`item_specifics` и перепарсиваются как новые.
- **Где код:**
  - `ebay_my_news/title_recheck.py` – вся логика проверки и удаления.
  - `ebay_my_news/main.py` – сбор `title` на этапе каталога и вызов проверки.
- **Тумблеры (.env):**
  - `RECHECK_TITLES=1` — включить режим.
  - `RECHECK_TITLES_LOG=1` — добавить инфо‑логи `title_changed …` (+ события в `excel_log`, если он активен).
- **Подключение к БД:** модуль использует `asyncpg` и подключается напрямую к Postgres. Параметры можно переопределить через `RECHECK_DB_HOST/PORT/NAME/USER/PASSWORD`, значения по умолчанию — `81.30.105.134:5421`, база `eb`, пользователь `admin/Password123`.
- **Порядок фильтров:** цена → блок‑лист продавцов → сравнение title. Только прошедшие первые два шага участвуют в проверке.
- **Удаление:** строгий матч по number; сначала `item_specifics`, затем `ebay`. Никаких temp-таблиц.
- **Перепарс:** после удаления объявление попадает в стандартный pipeline (`product()`), что гарантирует актуальные данные в БД и Redis Stream.

**Реестр примеров (для визуального анализа)**
- Каждый кейс расхождения заголовков сохраняется в `title_mismatch.log` (JSONL, по одной записи в строке).
- Путь можно переопределить переменной `RECHECK_TITLES_FILE`.
- Формат записи: `{ "number": ..., "old": ..., "new": ..., "query": ... }`.
